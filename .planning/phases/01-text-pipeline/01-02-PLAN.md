    1. Create yt_to_skill/llm/client.py with:

       - make_openai_client(config: PipelineConfig) -> OpenAI:
         Creates OpenAI client with base_url="https://openrouter.ai/api/v1" and api_key from config.

       - make_instructor_client(config: PipelineConfig):
         Wraps make_openai_client with instructor.from_openai().

       - translate_text(client: OpenAI, text: str, glossary: dict, config: PipelineConfig) -> str:
         Loads translate.txt prompt, injects glossary terms, calls chat.completions.create with config.translation_model and config.max_tokens_translation. Returns translated text. Wrapped with @retry(wait=wait_exponential(min=2, max=60), stop=stop_after_attempt(3), retry=retry_if_exception_type((APITimeoutError, APIConnectionError, RateLimitError))).

       - classify_content(client: OpenAI, title: str, description: str, transcript_sample: str, config: PipelineConfig) -> tuple[bool, float]:
         Loads filter_content.txt prompt, sends metadata + first 500 words of transcript. Uses config.filter_model. Returns (is_strategy: bool, confidence: float). Wrapped with @retry.

       - extract_trading_logic(instructor_client, translated_text: str, video_id: str, source_language: str, config: PipelineConfig) -> TradingLogicExtraction:
         Uses instructor_client.chat.completions.create with response_model=TradingLogicExtraction, temperature=0, config.extraction_model, config.max_tokens_extraction. Loads extract_trading.txt as system prompt. Wrapped with @retry.

       - load_glossary(glossary_path: Path) -> dict:
         Reads trading_zh_en.json, returns dict.

    2. All functions log model ID, stage name, and token usage via loguru after each call.

       - Mock the openai.OpenAI class to verify base_url and api_key

       - Test glossary injection by checking the messages content includes glossary terms

  <action>
    1. Create yt_to_skill/llm/prompts/ directory.

    2. Create translate.txt — system prompt for Chinese-to-English translation:
       - Instructs the LLM to translate a trading video transcript to English
       - Contains {glossary} placeholder where term pairs are injected at runtime
       - Rules: use glossary terms exactly, flag unknown trading terms as GLOSSARY_ADDITIONS at end of response, preserve segment structure, translate literally/technically (no paraphrasing trading concepts)
       - Handle non-Chinese transcripts: "If the transcript is already in English, return it unchanged. If it is in another language, translate to English."

    3. Create filter_content.txt — system prompt for non-strategy classification:
       - Instructs the LLM: "Given a video title, description, and transcript excerpt, determine if this video presents a specific actionable trading strategy with entry/exit rules."
       - Output format: "STRATEGY" or "NOT_STRATEGY" followed by confidence (0.0-1.0) and one-line reason
       - Strict threshold instruction: "When in doubt, classify as NOT_STRATEGY. Only classify as STRATEGY if there are clear entry conditions, exit conditions, or specific indicator setups discussed."

    4. Create extract_trading.txt — system prompt for structured extraction:
       - Instructs the LLM to extract trading logic from translated transcript
       - Defines each field: entry_criteria, exit_criteria, indicators, risk_rules, market_conditions, strategy_name
       - CRITICAL instruction: "If a parameter was NOT explicitly stated in the transcript, you MUST return null for that field. Do NOT infer, estimate, or assume values. A null value means the trader did not specify this parameter."
       - Instructs: "If the video describes multiple strategies or variations (e.g., aggressive vs conservative), create separate strategy objects in the strategies array."
       - Instructs: "Always populate raw_text with the verbatim translated excerpt for each condition, even when structured fields are filled."

       - Order types: 限价单/limit order, 市价单/market order, 止损/stop loss, 止盈/take profit, 挂单/pending order, 追踪止损/trailing stop
