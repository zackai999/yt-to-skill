  <action>
    1. Add all new test cases to `tests/test_cli.py`. Mock questionary calls (patch `questionary.checkbox`, `questionary.select`, `questionary.confirm`, `questionary.text`). Mock installer functions. Existing tests must continue to pass -- update them to use `yt-to-skill process <url>` syntax OR verify backward compat shim handles them.

    **Important for existing test compatibility:** The backward compat shim inserts "process" into sys.argv when the first arg looks like a URL. This means existing tests that use `["yt-to-skill", "https://..."]` should still work without modification. Verify this.

    **Important for test_install_conflict_offers_custom_name:** Mock `_check_conflict` to return True, mock `questionary.confirm().ask()` to return False (decline overwrite), mock `questionary.text("Enter a custom name for this skill:").ask()` to return "my-custom-name".     **Important for test_install_conflict_custom_name_cancel:** Same as above but mock `questionary.text().ask()` to return None. Assert `install_skill` is NOT called for that skill-agent pair.

    2. Refactor `yt_to_skill/cli.py`:

    **Subcommand structure:**
    - Top-level parser: `argparse.ArgumentParser(prog="yt-to-skill")`
    - `subparsers = parser.add_subparsers(dest="subcommand")`
    - `process` subparser: all existing flags (url, --output-dir, --force, --verbose, --no-keyframes, --max-keyframes) plus new `--install` flag (metavar="AGENTS", help="Comma-separated agent IDs to install to, skipping interactive prompt")
    - `list` subparser: no additional args
    - `uninstall` subparser: positional `name` arg

    **Backward compatibility shim (before parse_args):**
    ```python
    if len(sys.argv) > 1 and _is_url_like(sys.argv[1]):
        sys.argv.insert(1, "process")
    ```
    Where `_is_url_like(s)` checks `s.startswith(("http", "www."))`.

    **Fallback:** If `args.subcommand is None` (no subcommand, no URL), print help and exit 0.

    **_cmd_process(args):**
    - Contains existing batch loop logic (moved from main())
    - After batch loop, collect all successful skill paths: list of `(video_id, skill_dir_path)` tuples
    - If no successful skills generated, skip install flow
    - Call `_run_install_flow(skill_entries, args)` for the install step

    **_run_install_flow(skill_entries, args):**
    - Show skill summary for each entry: parse SKILL.md frontmatter, print name + strategy count + entry/exit count + keyframe count
    - If `args.install` is set (--install flag): parse comma-separated agent IDs, validate against known agents, install directly (no prompt). On conflicts, print warning and skip (require --force to auto-overwrite via flag path per Pitfall 6 from RESEARCH.md).
    - Else if `sys.stdin.isatty()` is False: print "(skipping install prompt in non-interactive mode)" and return
    - Else: interactive flow:
      1. `agents = detect_installed_agents()`
      2. If no agents detected: print "No Agent Skills-compatible tools detected" and return
      3. `selected = questionary.checkbox("Select agents to install into:", choices=agents).ask()`
      4. If user cancels (None) or selects none: return
      5. `scope = questionary.select("Install scope:", choices=["global", "project-local"]).ask()`
      6. For each skill_entry, for each selected agent:
         - Determine target path (global or project-local)
         - Sanitize skill name from frontmatter
         - Check conflict with `_check_conflict(target_path, sanitized_name)`:
           - If conflict exists: `overwrite = questionary.confirm(f"Skill '{name}' already exists in {agent}. Overwrite?").ask()`
           - If user confirms overwrite: `install_skill(src_dir, target_path, name, video_id, overwrite=True)`
           - If user declines overwrite: prompt for custom name per user decision:
             `custom = questionary.text("Enter a custom name for this skill:").ask()`
             If custom is None or empty string: skip this skill-agent pair, print "Skipping {name} for {agent}."
             Else: `custom_sanitized = sanitize_skill_name(custom)`, then `install_skill(src_dir, target_path, custom_sanitized, video_id, overwrite=False)`
         - If no conflict: `install_skill(src_dir, target_path, name, video_id, overwrite=False)`
      7. Print install summary table: skill name, agent, path, status (check/cross)

    **_cmd_list(args):**
    - Call `list_installed_skills()`
    - If empty: print "No yt-to-skill generated skills found.\n(only yt-to-skill generated skills shown)"
    - Else: print formatted table with columns: Name, Agent, Source Video, Installed At, Path

    **_cmd_uninstall(args):**
    - Call `uninstall_skill(args.name)`
    - If empty: print f"Skill '{args.name}' not found in any agent."
    - Else: print removed paths, one per line

    3. Run full test suite to verify no regressions.
